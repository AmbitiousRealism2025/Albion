Albion
GLM-5.2 orchestration, built around fable-mode

A lower-cost option for people who like Claude Code — without sacrificing too much. GLM-5.2 as the main model, fable-mode baked into a unified always-on operating system, deterministic hooks where Atreides trusted the model, pluggable vision, and a tmux Conductor that lets stock Claude Code orchestrate Albion workers. Grounded in an 11-agent research mission with live wire-level probes.

The intent gate & the inverted effort model

With fable-mode baked in, the gate routes depth and delegation, not skill activation. The main agent defaults to effort xhigh → GLM max (Z.ai's own coding recommendation); delegation downshifts. Three effective tiers (off / high / max), routed at task boundaries via agent frontmatter — never mid-session flips, never thinking keywords (verified: they don't reach the wire).

06

The hook enforcement suite

Destructive-command guard PreToolUse

permissionDecision: deny — blocks even in bypassPermissions. Atreides' normalization pipeline ported verbatim, finally wired in. Hard floor: modern-syntax permissions deny-list.

Strike counter PostToolUse

Counts per-operation failures in session-state JSON; injects "Strike 2 of 3" as factual context. Strike 3 → counterexample-first recovery; git-revert demoted to escalation.

Completion gate Stop

Blocks "done" while tasks are open, the last test failed, or verification.md is empty on fable tasks. Loop-guarded, capped at 4 blocks, reads last_assistant_message for evidence-free-claim heuristics. The conversion aimed at GLM's reward-hacking lineage.

Secrets scrubber PostToolUse · workbench writes

updatedToolOutput redaction on .agent-workbench/** — fable-mode's advisory note becomes mechanical.

State re-injection SessionStart · compact|resume|clear

The verified compaction-survival channel. Re-injects task.md + state-map.md + strike/task state, each under the 10k-char cap.

Image-read interception PreToolUse · Read on images

Routes image reads to the vision subsystem and returns the description via updatedToolOutput — GLM-5.2 never receives raw image content it may mishandle. Degrades gracefully when unconfigured.

Anti-inert discipline albion doctor + CI

Every hook integration-tested with recorded real payloads; doctor re-runs the suite on installed hooks; CI runs it on every commit. Atreides had 466 tests and zero against the wire format.

07

Vision & the Conductor

Pluggable vision providers albion-vision

GLM-5.2 is powerful but not a vision model. Default: GLM-4.6V via the same Z.ai key (zero extra setup). Configurable alternates: Claude, Gemini, OpenAI — or orchestrate out to an external vision-capable agent session via the Conductor protocol. Three levels: one-shot helper CLI, transparent image-read interception, interactive vision session.

The Conductor stock CC ↔ Albion · tmux

A skill that works in both stock Claude Code and Albion: Fable does architecture and review at frontier prices, sparingly; GLM-5.2 does volume implementation at ~1/5 cost. tmux for transport and observability, files for signaling — the fable workbench is the wire format, and the worker's Stop-gate hook writes the completion manifest. Fire-and-forget baseline; attachable live; harness-neutral protocol. Albion never requires it: standalone strength is the product.

Roadmap

  • M0

    Verification matrix

    Filled by live research probes. Remaining: cache-keying on effort, subagent prompt-accounting, current Pro/Max pricing.

  • M1

    Launcher + doctor

    albion / albion --vanilla / claude modes; startup validation fails loudly on wrong endpoint, silent model remap, or version < 2.1.163; synthetic payloads through every hook.

  • M2

    Hook suite + session-state engine

    Five hooks with recorded-payload tests. Exit: Stop gate blocks a fake "done"; strike counter injects at 2; scrubber redacts a planted secret.

  • M3

    ALBION.md + skills + agents, packaged as a plugin

    Unified ~600-line operating system (fable-mode baked in); manifest→compile pipeline; 4 crown-jewel skills; 5 agents. Exit: a long-horizon task runs the full loop end-to-end solo; fable-mode also usable standalone in stock CC.

  • M4

    Vision + Conductor

    Provider registry (GLM-4.6V default, Claude/Gemini/OpenAI/external alternates), albion-vision helper, image-read hook; conductor skill with the completion-manifest protocol. Exit: a Fable session dispatches an Albion worker and reviews its manifest.

  • M5

    Telemetry + bench (continuous from M2)

    Dual cost model (prompt-equivalents on plan, dollars on API); three-arm A/B finally answers whether fable-mode measurably improves GLM-5.2's long-horizon discipline.

  • M6

    OSS release 1.0 — the pristine repo

    MIT, CONTRIBUTING, SECURITY, templates, CI running the recorded-payload hook tests, semver releases, fresh-machine quickstart ≤5 min, data-sovereignty disclosure. Grant-submission-ready.

  • M7

    Hardening

    Provider abstraction with data-sovereignty toggle; interactive conductor steering; lessons-promotion flow; bench green across ≥2 providers. Then: Oakdale (Opencode) and Bower Lake (Pi) ports over the same manifest and conductor bus.

10

Tensions the design resolves — and how

  1. Trigger authorityDissolved in v0.2: fable-mode is baked into ALBION.md, so there is nothing to trigger. The gate now routes only depth and delegation.
  2. Hooks vs promptsHooks check existence and state; the skill owns honesty; the verifier agent is the semantic layer. Every rule assigned to exactly one substrate.
  3. Compliance vs coverageGLM needs more structure but obeys long lists less. The <400-line charter is a hypothesis — A/B compliance probes live in the bench.
  4. Two state systemsModel writes prose (workbench); hooks write JSON (session-state); JSON wins on disagreement.
  5. Recovery philosophyCounterexample-first is primary; git-revert is the escalation after repeated counterexample-loop failure. The hook counts, the model reasons.
  6. Bounded autonomySWE-Marathon says decompose: short verified segments, hook checkpoints, re-anchoring injections — 1M context is headroom, not a working set.
  7. Unvalidated foundationsBuild the harness, instrument the metrics, run the three-arm experiment continuously rather than gating the build on a lab study.
  8. Data sovereigntyProvider toggle surfaced per-project; the regression bench runs per-provider because quantized hosts can silently degrade compliance behavior.
